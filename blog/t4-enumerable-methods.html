Week 4 Phase 0

Enumerable Method

3/21/15

This week I am going to teach you about the ruby method ".cycle" The first thing we to do is to take a look at the definition on Ruby docs.
The first part of the definition looks like this:
cycle(n=nil) {|obj| block} -> nil
The first part is the .cycle method being called. The (n=nil) is saying that if you don't give the cycle a number, by default n is equal to nil. If cycle doesn't have a n given but does have a block, the cycle will run the block forever because you aren't telling it how many times to cycle. Otherwise, whatever you provide for n, cycle will run the block that many times. If n is a negative number, the block is not run and nil is returned. {|obj| block} is saying that we create a temporary object and run the block n times. Another thing to note that if you run .cycle on an empty object, like an array with nothing in it, nil will be returned as well. So really what cycle does is iterate through an array n times. Lets take a look at a few examples of what we just went over:

a = ["a","b","c"]
a.cycle {|x| puts x} => will print a,b,c forever!
a.cycle(0) {|x| puts x} => will print nothing and return nil
a.cycle(1) {|x| puts x} => will print a b c and return nil

So lets take a look at what is happening here.

  1. The first thing we did was create a new array, a, and set the attributes in it to a,b,c.
  2. The first example we ran cycle on our array but with no n value. Therefore the block will just run forever because we never told it how many times to actually run.
  3. The second example we set our n to 0 so we aren't actually going to cycle through anything so the only thing that happens is that nil is returned.
  4. In the final example, we set n to 1 so cycle will run the block one times, which is to print(puts) our array a.

Before we get to the second half of the definition, a few key notes to take away from here:

  1. Cycle will iterate over the array
  2. n by default is nil
  3. If the loop finishes without getting interrupted, nil is returned
  4. Cycle saves elements in an internal array, so changes after the first pass have no effect on the original object

The second part of the definition looks like this:
cycle(n=nil) -> an enumerator
What this is saying is that if we run .cycle on an object, an enumerator is returned. Without getting into too much detail, an enumerator is a class that allows both internal and external iterations. I will not go into too much detail because there are many different things you can do within the Enumerator class. So just to touch on it real quick, the object becomes an enumerator and we can run other methods within the enumerator class by setting it equal to a variable such as cycle_enum = a.cycle, then running something like
cycle_enum.next which will run .next on each element of our original array.
